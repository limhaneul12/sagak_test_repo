=== 식품 영양성분 데이터베이스 컬럼 매핑 방안 ===

원본 엑셀 파일 구조 분석 결과 다음과 같은 매핑이 필요합니다.

1. 요청인자.md에 명시된 출력 항목과 원본 데이터 컬럼 매핑:

| 요청인자.md 출력항목 | 원본 엑셀 컬럼명         | 설명                  |
|---------------------|------------------------|----------------------|
| id                  | NO                     | 자동 생성되는 번호       |
| food_cd             | 식품코드                | 식품 고유 코드          |
| group_name          | 식품대분류              | 식품군 분류             |
| food_name           | 식품명                  | 식품 이름              |
| research_year       | 연도                    | 조사년도 (YYYY)        |
| maker_name          | 지역 / 제조사            | 지역 또는 제조사 정보    |
| ref_name            | 성분표출처              | 자료 출처              |
| serving_size        | 1회제공량               | 1회 제공량(숫자)        |
| calorie             | 에너지(㎉)              | 열량(kcal)            |
| carbohydrate        | 탄수화물(g)             | 탄수화물(g)            |
| protein             | 단백질(g)               | 단백질(g)              |
| province            | 지방(g)                 | 지방(g)                |
| sugars              | 총당류(g)               | 총당류(g)              |
| salt                | 나트륨(㎎)               | 나트륨(mg)             |
| cholesterol         | 콜레스테롤(㎎)            | 콜레스테롤(mg)          |
| saturated_fatty_acids | 총 포화 지방산(g)       | 포화지방산(g)           |
| trans_fat           | 트랜스 지방산(g)         | 트랜스지방(g)           |


2. 데이터 처리 방안:

   a. 컬럼명 변환:
      - 원본 데이터의 한글 컬럼명을 영문 컬럼명으로 변환
      - 공백 및 특수문자 처리에 주의

   b. 데이터 타입 변환:
      - 숫자형 데이터(serving_size, calorie 등)는 float 타입으로 변환
      - 문자열 데이터는 string 타입으로 유지
      - 원본의 '-' 값은 None/NaN으로 처리 후 적절히 변환

   c. 데이터 정제:
      - 숫자형 필드의 단위 제거 (예: '500g' → 500)
      - 결측치 처리: 숫자형은 0, 문자열은 빈 문자열('')로 대체

3. 최종 출력 형식:
   - CSV 형식으로 출력하되, UTF-8 인코딩 사용
   - 헤더는 영문 컬럼명 사용
   - 모든 필드는 요청인자.md에 명시된 순서대로 배치

4. 추가 작업:
   - SQL 데이터베이스 삽입을 위한 INSERT 문 자동 생성
   - 배치 처리로 대용량 데이터 효율적 처리
